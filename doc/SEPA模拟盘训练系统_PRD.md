## 0. 一句话定义

> 一个**强制纪律执行**的 SEPA 模拟盘训练系统：用模拟资金跑 Minervini 策略，系统逐笔卡住"市场闸门→选股模板→买点检查→止损挂单→复盘"全流程，**核心产出是纪律达标率，不是收益率**。

---

## 1. 背景与目标

### 1.1 背景
- 用户拟用真金（¥30万实验仓）执行 SEPA 趋势交易，但 90% 散户死于"不执行止损"。
- 在投入真钱前，需用模拟盘验证：**能否机械地执行系统、扛住连续止损不变形**。
- 现有手册是静态文档，缺乏强制执行与数据沉淀工具。

### 1.2 目标（按优先级）
| 优先级 | 目标 | 衡量 |
|---|---|---|
| P0 | 强制走完每笔交易的规则闭环 | 无法跳过止损设置即下单 |
| P0 | 自动统计纪律 KPI | 纪律达标率 / 盈亏比 / 胜率 / 样本量 |
| P1 | 辅助选股（趋势模板校验） | 输入标的→8条逐条判定 |
| P1 | 复盘沉淀 | 每笔可记录、可回看、可标注违纪 |
| P2 | 行情数据自动化 | 减少手工录入 |

### 1.3 非目标（明确不做）
- ❌ 不做真实下单 / 券商对接（纯模拟）
- ❌ 不做策略自动交易 / 量化回测引擎
- ❌ 不追求收益最大化，不做荐股
- ❌ 不做多用户 / 社交（单用户自用）

## 3. 功能架构总览

```
SEPA 模拟盘训练系统
├── M1 市场闸门监测      （总开关：能不能开新仓）
├── M2 股池/Watchlist    （候选股 + 8条趋势模板校验）
├── M3 交易决策助手      （买点检查清单 + 止损位计算）
├── M4 模拟持仓管理      （仓位、止损单、实时盈亏）
├── M5 交易日志与复盘    （逐笔记录 + 违纪标记）
├── M6 KPI 仪表盘        （纪律达标率/盈亏比/胜率/样本量）
└── M7 纪律强制引擎      （贯穿各模块的硬性拦截规则）
```

## 4. 模块需求详述

### M1 · 市场闸门监测

**作用:** 判断当前能否开新仓（手册第二章）。

**功能点:**
- 录入/展示 4 项闸门指标状态：
  1. 标普500 & 纳指 是否站上 50日均线
  2. 50日均线斜率是否向上
  3. 市场宽度（涨跌家数比 > 1）
  4. 新高 vs 新低家数
- 闸门状态汇总为 **开启 / 关闭** 二态
- 闸门关闭时，M3 的"开新仓"动作被禁用（仅允许管理已有持仓）

**用户故事:**
> 作为交易者，当大盘趋势走坏时，我希望系统直接禁止我开新仓，这样我不会在弱势中乱开仓。

**验收标准:**
- [ ] 闸门关闭状态下，新建买入单按钮置灰且提示原因
- [ ] 闸门状态变更有时间戳记录

---

### M2 · 股池 / Watchlist

**作用:** 维护候选标的，逐条校验 8 条趋势模板（手册第三章）。

**功能点:**
- 添加/删除标的（代码 + 名称）
- 对每只标的录入或拉取关键数据：现价、50/150/200日均线、52周高/低、RS 值
- 系统自动对照 **8 条趋势模板** 逐条判定 ✅/❌，给出"通过 / 未通过"结论
- 基本面加分项（EPS增速、营收加速、催化剂）可手动标注
- 支持标记 VCP 形态阶段（收缩次数、是否接近枢轴）

**8条趋势模板判定逻辑（开发须严格实现）:**
| # | 条件 |
|---|---|
| 1 | 现价 > 150日均线 且 > 200日均线 |
| 2 | 150日均线 > 200日均线 |
| 3 | 200日均线连续上升 ≥ 1个月 |
| 4 | 50日均线 > 150日均线 且 > 200日均线 |
| 5 | 现价 > 50日均线 |
| 6 | 现价距52周低点 ≥ 25% |
| 7 | 现价距52周高点 ≤ 25% |
| 8 | RS ≥ 70 |

**验收标准:**
- [ ] 8条全过才显示"✅ 入池"，否则列出未通过的具体条目
- [ ] 可按"是否通过""距枢轴距离"排序

---

### M3 · 交易决策助手（核心模块）

**作用:** 下单前强制走完检查清单，自动算止损位与最大风险（手册第五、六章）。

**功能点:**
- **买入前检查清单**（逐项打勾，全勾才能下单）：
  - [ ] 市场闸门开启
  - [ ] 8条趋势模板通过
  - [ ] VCP 形态确认（收缩次数、波动递减）
  - [ ] 当前价距枢轴 ≤ 5%
  - [ ] 已设定止损价
- **自动计算:**
  - 输入：买入价、止损价(或默认-7%)、单笔仓位金额
  - 输出：止损幅度%、最大亏损金额、占总仓比例
  - **校验：最大亏损 > 总仓1.5%（≈$600）时 → 红色警告并阻止下单**
- 下单时**强制同时生成止损单**（无止损价不可提交）

**用户故事:**
> 作为交易者，我希望系统在我下单前算好"这笔最多亏多少"，超限就拦住我，这样我永远不会单笔下重注。

**验收标准:**
- [ ] 检查清单未全勾，提交按钮不可用
- [ ] 未填止损价，无法提交订单
- [ ] 最大亏损超 1.5% 触发硬拦截

---

### M4 · 模拟持仓管理

**作用:** 管理模拟资金、持仓、止损单、盈亏。

**功能点:**
- 账户总览：初始资金¥30万、当前市值、现金、累计盈亏、距熔断线(-15%)还有多少
- 持仓列表：代码、买入价、现价、止损价、浮动盈亏%、距止损距离
- **止损触发提醒:** 现价触及止损价 → 强提醒（手册要求无条件执行）
- **熔断提醒:** 账户总亏损达 -15% → 全局警告"强制停手复盘"
- 加仓/减仓/平仓操作（每次操作进入 M5 日志）

**止损/止盈规则（系统应内置提示，手册第六、七章）:**
- 硬止损 -7~8%
- 时间止损：买入5日内未涨2% → 提示考虑离场
- 盈利20% → 提示可减半 + 止损上移到成本
- 跌破50日线 → 提示减半/清仓

**验收标准:**
- [ ] 触及止损价有醒目提醒（颜色/弹窗）
- [ ] 熔断线触发全局拦截新仓 + 提示复盘
- [ ] 涨跌颜色遵循 A股习惯（涨红跌绿）

---

### M5 · 交易日志与复盘

**作用:** 逐笔记录，沉淀数据，标记违纪（手册第九章）。

**字段（数据模型见第6节）:**
- 自动带入：日期、代码、方向、价格、股数、金额、枢轴点、止损价、最大风险$
- 手动填写：买入理由、卖出理由、复盘备注
- **关键字段:「是否按规则止损」✅/❌** ← KPI 计算核心

**功能点:**
- 交易流水列表（可筛选：盈利/亏损、是否违纪、时间段）
- 单笔详情页：进出场点位 + 盈亏 + 复盘文字
- 违纪交易高亮（未按规则止损的标红）
- 周末复盘清单（提示更新watchlist、回顾本周违纪）

**验收标准:**
- [ ] 每笔平仓后强制要求标注"是否按规则止损"
- [ ] 可导出 CSV

---

### M6 · KPI 仪表盘

**作用:** 把训练成果量化为可判定的指标（手册第十章）。

**核心 KPI（开发须精确计算）:**
| KPI | 计算公式 | 目标值 | 展示 |
|---|---|---|---|
| 纪律达标率 | 按规则止损笔数 ÷ 应止损笔数 | 100% | 大数字 + 进度环 |
| 盈亏比 | 平均盈利% ÷ 平均亏损% | ≥ 2 | 数字 + 趋势 |
| 胜率 | 盈利笔数 ÷ 总笔数 | ≥ 35%（参考） | 数字 |
| 样本量 | 已完成交易笔数 | ≥ 15–20 | 进度条 |
| 总收益 | 当前市值 ÷ 初始资金 - 1 | 不设要求 | 数字（参考） |

- **实验评估结论区:** 满足"纪律100% + 盈亏比≥2 + 样本≥15" → 显示"✅ 通过，可加码至¥50-80万"；否则显示"继续训练"
- 时间维度：可看本周/全程

**验收标准:**
- [ ] 纪律达标率实时随交易日志更新
- [ ] 各 KPI 与目标值对比有视觉区分（达标/未达标）

---

### M7 · 纪律强制引擎（横切规则）

**作用:** 这是本系统区别于普通记账工具的灵魂——**用产品逻辑替用户管住手**。

**硬性拦截规则汇总:**
| 场景 | 拦截动作 |
|---|---|
| 市场闸门关闭 | 禁止开新仓 |
| 8条模板未全过 | 禁止加入"可买入"状态 |
| 买入检查清单未全勾 | 提交按钮置灰 |
| 未设止损价 | 无法提交订单 |
| 单笔最大亏损 > 1.5% | 红色拦截 |
| 距枢轴 > 5% 追高 | 警告（可强制确认后放行，记录为风险操作） |
| 账户亏损达 -15% | 全局禁止开新仓 + 强制复盘提示 |
| 平仓未标注是否违纪 | 不允许关闭该笔 |

**心理提醒卡（手册附录B，可在关键节点弹出）:**
- 下单时："我是来执行系统的，不是来证明自己对的。"
- 触及止损时："止损不是失败，是系统正常运转。"
- 连续止损3次："检查闸门是否该关。"

## 5. 关键业务流程

### 5.1 开新仓主流程
```
查看M1闸门 → 闸门开?
  ├─ 否 → 禁止，仅管理持仓
  └─ 是 → 从M2选已入池标的
          → M3填买入价/仓位
          → 系统校验检查清单+最大风险
          → 全部通过? 
              ├─ 否 → 拦截并提示
              └─ 是 → 生成持仓+止损单(M4) → 写入日志(M5)
```

### 5.2 平仓主流程
```
触发卖出(止损/止盈/趋势坏)
  → 记录卖出价/盈亏
  → 强制标注"是否按规则止损"
  → 写入日志(M5)
  → 更新KPI(M6)
```

## 7. 非功能需求

| 类别 | 要求 |

| 平台 | Web 优先（本地运行即可），响应式兼顾移动查看 |

## 8. MVP 范围与迭代规划

### MVP（第一版，必做）
聚焦"纪律强制 + 数据沉淀"，行情数据**手动录入**即可：
- ✅ M3 交易决策助手（检查清单 + 止损/风险计算 + 拦截）
- ✅ M4 模拟持仓管理
- ✅ M5 交易日志（含违纪标记）
- ✅ M6 KPI 仪表盘
- ✅ M7 纪律强制引擎
- ⚪ M1 闸门：简化为手动开关
- ⚪ M2 股池：手动录入数据 + 自动判定8条

### V2（增强）
- M2 股池数据自动拉取（API）
- M1 闸门指标自动计算
- 周末复盘提醒、连续止损预警

### V3（可选）
- 历史交易统计图表（盈亏分布、持仓周期分析）
- 移动端优化

**MVP 优先级排序:** M3 + M7 > M5 + M6 > M4 > M2 > M1

## 9. 风险与注意

| 风险 | 说明 | 应对 |
|---|---|---|
| 自己绕过拦截 | 系统是自律工具，用户可改数据作弊 | 把"违纪"也记录下来，复盘时面对真实数据 |
| 行情数据准确性 | 手动录入易错 | V2 接 API；MVP 阶段录入后二次确认 |
| 模拟≠实盘心理 | 模拟盘没有真金的痛感 | KPI 看的是"执行动作"而非情绪，可迁移 |
| 过度设计 | 功能堆砌反而不用 | 严守 MVP 范围，先跑起来 |

## 10. 验收总标准

系统达成以下即视为可用：
1. 能完整走通"开仓→持有→平仓→复盘→KPI更新"闭环
2. 七条硬性拦截规则全部生效
3. 纪律达标率等4个核心KPI能准确计算并展示
4. 跑满一个3个月实验周期后，能据 KPI 给出"是否通过、能否加码"的明确结论
