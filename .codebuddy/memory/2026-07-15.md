## SEPA 输入框文字被上下裁切修复
- 现象：SEPA 决策助手所有 `.sepa-input`（uni `<input>`）里的 placeholder/文字上下被裁切。
- 根因：uni-app H5 的 `uni-input` 元素默认 `height:1.4em; overflow:hidden`；`.sepa-input` 之前只设 `min-height:44rpx` 未覆盖 `height`，外层高度被钉在 1.4em(~39rpx)，叠加 box-sizing:border-box + 上下 padding 16rpx(共32rpx)后内容区几乎为 0 → 文字被 overflow:hidden 裁切。
- 修复 `frontend/src/pages/sepa/sepa-shared.css` 的 `.sepa-input`：加 `height:auto`、`overflow:visible`，`min-height` 44→76rpx（=行高42+padding32+border2）；内层选择器补上 `.uni-input-placeholder`，加 `overflow:visible`、`min-height:42rpx`。build 通过。
- 通用坑：uni H5 input 高度调大务必显式 `height:auto`（覆盖默认 1.4em），否则 min-height 撑不开会被 overflow:hidden 裁切。

## SEPA VCP 形态自动识别（P1+P2+HK 数据源，本次实施）
- **用户决策**（4 条）：①阈值先宽松（回测后再收紧）②人在环上（算法判+用户拨 `vcp_confirmed` 开关，不自动放行）③先做 P1(算法+接口)+P2(回测) ④HK 也要补历史日K。
- **核心算法** `backend/app/services/vcp_detector.py`（纯 OHLCV、DB 无关、可回测）：ZigZag 拐点检测(收盘定转向、回填真实 high/low)→ 以窗口内最高 swing high 为枢轴 → 往右配对 (高→低) 收缩段 → 五项硬指标：收缩次数 2~5、振幅逐次递减(≤上段90%，允许1例外)、末段振幅≤8%、量能逐段递减(≤95%，允许1例外)。**关键修正**：经典 VCP 末次回升逼近枢轴会略高于前高，故"高点递减"仅作信息字段、不作硬门槛（核心是振幅/量能递减）。`detect_vcp(bars, params?, pivot_override?)` 返回 `{vcp_detected, contractions, amplitudes, vol_decays, decay_ok, vol_decay_ok, high_ok, near_pivot, pivot_suggested, segments, swing_points, ...}`。
- **宽松初值常量**（回测后切换严格值，注释在文件内）：ZIGZAG_THRESHOLD=2%(严3%)、MAX_CONTRACTIONS=5(严4)、AMP_DECAY_RATIO=0.90(严0.85)、LAST_AMP_MAX=8%(严5%)。
- **接口** `backend/app/api/v1/sepa.py` 新增 `GET /sepa/vcp/analyze`（公开，API Key）：`market,symbol,pivot_price?` → 查 `stock_daily_quote` 近 120 根 → `detect_vcp`，不写库。HK 代码 `zfill(5)`。无历史数据返回 `data_available=False`。人在环上由既有 `vcp_confirmed`(CheckRequest/TradeOpen 请求体) 兜底。
- **HK 历史日K** `backend/app/services/hk_data_fetcher.py`：腾讯 `hkfqkline`(主)+新浪港股 K 兜底 → upsert `stock_daily_quote(market='HK', ts_code=5位零填充)`。`refresh_hk_quotes()` 抓 SEPA 股池 HK 标的（幂等）。调度器 `scheduler.py` 新增 `_hk_quotes_job`（每交易日 16:30 北京时间，HK 收盘后）。回补脚本 `backend/scripts/backfill_hk_kline.py --codes 00700,09988`。
- **回测脚本** `backend/scripts/backtest_vcp.py`：滑动窗口跑 `detect_vcp`，统计命中后 5/10/20/60 日收益分布 vs 随机买入基准，CSV 导出 + 阈值敏感性扫描(`--sensitivity`)。运行：`docker compose run --rm -v /home/Alphareader/backend:/app web python scripts/backtest_vcp.py --market ALL --max-symbols 200 --step 5`。
- **测试** `backend/tests/test_vcp_detector.py`（5 passed）：VCP识别/上升趋势拒/等幅震荡拒/数据不足拒/pivot覆盖。
- **待续（用户未要求本次做）**：P3 前端判定卡片 + K线缩略图标注；P4 `refresh-vcp` 批量回填（届时给 `SepaWatchlistItem` 加 `vcp_auto` JSON 列承载算法结果，不改 `vcp_confirmed` 人工字段）。
