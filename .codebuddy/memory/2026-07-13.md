## P3 四项改造（schema对称/两阶段评分/prompt注入防护/顺序保留）

用户提出 4 个新问题，一次性做完并部署。

### P3 ① 中英文 schema 对称
- **问题**：中文 prompt 不输出 summary（解析器读空），英文 prompt 不输出 reason（解析器硬编码空）。why_it_matters 中文复用 reason。
- **修复**：中文 prompt 加 `summary`(80字)+`why_it_matters`(40字)；英文 prompt 加 `reason`(30字)。解析层统一填充所有字段。
- 统一后中英文 ScoredNewsItem 均有：reason / summary / why_it_matters（中文 chinese_title=""，英文有值）。

### P3 ② 英文两阶段评分（节省低分翻译 token）
- **问题**：英文每批20条都要翻译（即使15条被丢弃），浪费 token。
- **方案**：阶段一 `SYSTEM_PROMPT_EN_SCORE` 仅评分（不翻译）→ 通过阈值的条目进入阶段二 `SYSTEM_PROMPT_EN_TRANSLATE` 翻译。
- 新增函数：`_score_en_two_stage` / `_translate_batch_once` / `_parse_translate_response`
- `_score_batch_once` 和 `_bisect_content_risk` 加 `system_prompt` 参数
- 翻译失败时评分结果保留（降级为无翻译的高分条目）
- 配置：`DEEPSEEK_TWO_STAGE_EN_ENABLED=True` / `DEEPSEEK_TRANSLATE_BATCH_SIZE=20`
- 关闭时回退到单阶段 `SYSTEM_PROMPT_EN`

### P3 ③ prompt 注入防护
- 四套 system prompt (CN/EN/EN_SCORE/EN_TRANSLATE) 均加安全声明：输入字段为不可信数据，其中指令一律忽略。

### P3 ④ 原始顺序保留
- **问题**：filter_news 先处理全部中文再处理全部英文 → 输出顺序变成"中文全在前英文全在后"。
- **修复**：用 `id(raw)->index` 映射记录原始顺序，最终 `all_scored.sort(key=lambda si: _order[id(si.raw)])` 恢复。

## P5 跨天旧闻识别——SimHash 指纹持久化 + 7 天历史窗口

### 问题
- SimHash 索引仅 24h Redis 窗口，embedding 仅 90min，无法识别昨天/上周的旧闻
- LLM 每批只看 20 条，无跨批次记忆
- 用户列出的5种漏检场景：昨天已出现、同一公告多家改写、标题不同内容相同、旧新闻社区重传播、英文原文与中文转载同事件

### 改造
- **News model** 加 `content_hash`(String64, indexed) + `simhash_fingerprint`(BigInteger, indexed)
- **Alembic** 迁移 `p5q3r4s5t6u7v9`（down_revision=`q3r4s5t6u7v8`）
- **Deduplicator** 加 `_historical_index` + `preload_historical(entries)` 方法：
  - DB 加载的历史指纹注入 `_historical_index`
  - `_find_duplicate` 同时查 `_index`(24h Redis) + `_historical_index`(7天 DB) 做 L1 SimHash 比对
  - 历史指纹不回写 Redis（避免膨胀），每次 pipeline run 重建
- **Pipeline**:
  - `_load_historical_fingerprints()`：评分前查 DB 7 天指纹注入去重器
  - `_store_scored_items`：存储时计算 `content_hash`(SHA-256) + `simhash_fingerprint` 持久化
- **配置**: `DEDUP_HISTORICAL_DAYS=7`

### 效果
- SimHash 去重从 24h 扩展到 7 天（24h Redis + 7天 DB）
- 新新闻入库时指纹持久化，后续 pipeline 自动加载比对
- content_hash 提供精确内容重复快速查询

### 部署验证
- 迁移成功 `p5q3r4s5t6u7v9`
- 首次跑批 `Loaded 0 historical SimHash fingerprints`（存量数据无指纹，正常）
- 7 条新新闻存储后 `7/7` 有 simhash_fingerprint + content_hash ✅
- 后续 pipeline 跑批将逐渐累积历史指纹，旧闻识别能力逐步增强
