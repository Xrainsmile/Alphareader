### P0：核心健壮性（覆盖 ①④⑥⑦⑧⑨）
- `BatchStatus = Literal["ok", "api_error", "parse_error", "content_risk", "empty_after_filter", "no_api_key"]` + `BatchResult` dataclass，替代"失败就返回空 list"的老语义。
- **完整性校验** `_parse_response_detailed()` 返回 `(scored, processed_ids, missing_ids, duplicate_ids, parse_ok)`；missing>20% 会触发重试。
- **字段严格校验** `_ensure_string_list / _validate_ticker / _clean_tickers`。ticker 正则最终定为 A股6位/港股5位（4位自动补0）/**美股3-5位纯字母，可选 .X 后缀**（`^[A-Z]{3,5}(\.[A-Z])?$`）。测试用例"XX"、"invalid"必须被拒。
- **中文占比** `_chinese_ratio` = CJK / (CJK+ASCII letters)，取代"只查一个汉字"。阈值可配（title ≥0.5 / summary ≥0.6，见 `DEEPSEEK_MIN_CHINESE_RATIO_*`）。
- **langdetect 稳定化** `DetectorFactory.seed = 0`，字符占比优先，langdetect 兜底。
- **正文预览 200 → 800**（`DEEPSEEK_CONTENT_PREVIEW_CHARS`），prompt 加入 published_at + fetched_at 供 LLM 判定旧闻。
- **prompt 收紧**："发布时间距抓取时间超过 24h 或正文明确提及 3 天前以上一律判旧闻，最高 3 分"、"预期差需明文数字或 beat/miss 字样，仅凭语气不算"。
- **向后兼容**：`_parse_response()` / `filter_batch()` 保留为薄包装。
- **重要小坑**：test_parser.py 的 `mock_settings` 用 MagicMock，未定义的新 setting 会返回 MagicMock 对象参与 `float(...)` 比较报错。解决：解析代码里用 `getattr(settings, "X", default)` + `isinstance` 兜底，不改测试。

### P2：两层筛选 is_highlight（覆盖 ③）
- 采用**方案 B**：模型显式输出 `is_highlight: bool`，与 ai_score 解耦。
- Prompt 硬约束：`is_highlight=true` 必须同时满足 ① score≥8 ② 明确强催化 ③ 明文量化数据 ④ 一周内新闻。
- 解析层**硬防线**：即便模型返回 `is_highlight=true` 但 score<8，代码强制降级为 false。
- 类型宽容：接受 `bool` / `"true"/"false"` 字符串，其它类型（int/dict/None）安全降级为 false。
- DB schema：`news` 表加 `is_highlight BOOLEAN NOT NULL DEFAULT false`，带索引。迁移 `q3r4s5t6u7v8_add_is_highlight.py`（down_revision=`p2q3r4s5t6u7`）。
