## 首页新闻空白最终修复（onShow → onMounted）

继 7/8 的 onShow 修复（commit 45ef08c）后，用户反馈 reports 页正常但首页 news 仍空白。

**对比定位**：
- `pages/reports/index.vue` 用 Vue 标准 `onMounted` 触发首次加载 → H5 下可靠触发 → 数据正常。
- `pages/index/index.vue` 用 uni-app `onShow`（7/8 改的）→ **onShow 在 H5 首次加载时机不可靠，不触发首次加载** → 首页空。
- 验证部署产物确实含 7/8 的 onShow 修复（chunk 从 `uni-app.es.*.js` 导入 4 个钩子），证明问题在 onShow 本身不触发，而非没部署。

**修复（commit cb49be0）**：`frontend/src/pages/index/index.vue`
- vue import 加回 `onMounted`。
- 首次加载改用 `onMounted(() => { initTracker(); _initialLoaded=true; doResetAndLoad()... })`（与 reports 页一致，H5 必触发）。
- `onShow` 仅负责「离开后返回」的按需重载/滚动恢复；首次会话（`_lastHideTime===0`）直接 return，避免与 onMounted 重复加载。
- `onHide/onPullDownRefresh/onReachBottom` 保持 uni-app 官方钩子不变。

**部署验证**：服务器 git pull → `docker compose up -d --build frontend`。新 chunk `pages-index-index.CB2BVS0k.js`（旧 `D_FYULYd`），import 行比旧的多 `z as $`（即新导入的 onMounted），证明修复进入产物。

**关键经验**：uni-app H5 下，页面**首次加载不要依赖 `onShow`**，用 Vue 标准 `onMounted` 最可靠（reports 页是成功参照）。`onShow` 仅适合「返回时按需刷新」场景。minify 会重命名导入的钩子标识符，验证部署产物要看 import 行符号变化或模块路径（`uni-app.es.*.js` / 从 vue runtime 多导入符号），不能 grep `onShow(` 字符串。

## ⚠️ 重要更正：首页空白真正根因是内层 template（不是生命周期）

上面 onShow→onMounted 的结论是**误判**。用 playwright 实测线上首页发现：卡片 DOM 一直存在（cardCount=20、数据完整、computed display:block/visible），但 `offsetWidth/Height=0`。沿 DOM 祖先链定位：`.news-list` 的父是一个**真实 `<template>` HTML 元素，`display:none`**，把整个新闻区裹死。

**真正根因**：`pages/index/index.vue` 模板里有一个**无条件的内层 `<template>`**（仅作分组用），被 uni-app H5 编译器渲染成了真实 HTML `<template>` 元素（HTML 规范 `<template>` 默认 `display:none`，内容不渲染）。外层带 `v-if` 的 `<template>` 是 fragment（不产生 DOM），但无指令的内层 `<template>` 被当真实元素。

**修复（commit bf39cd3）**：删除内层无条件 `<template>` 及其闭合标签，让 NewsFilterPopover/news-list/卡片直接落在外层 `<template v-if="!searchMode">` fragment 下。playwright 复测：cardOffsetH 237、listOffsetH 4284、bodyLen 3768、`document.querySelector('template')` 为 null，新闻完全可见。

**反思**：生命周期（onShow/onMounted）从头到尾都正常触发——卡片一直渲染了、数据一直加载了，只是被 `display:none` 的 template 藏住。之前两轮"修复"（onShow 官方钩子、onMounted）都没必要，纯属误判方向。教训：页面"空白"应**先用浏览器 DevTools 看 DOM 是否存在 + offsetWidth/Height**，区分"没渲染"vs"渲染了但不可见（布局/CSS）"，而不是先怀疑数据/生命周期。`<template>` 在 uni-app H5 下无指令会被当真实元素，**模板里不要用裸 `<template>` 做分组**，用 `<view>` 或直接去掉。
